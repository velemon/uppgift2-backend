Om denna webbplats

Denna webbplats är skapad som en fristående frontend-applikation som konsumerar ett REST-baserat API utvecklat i en separat backend-del. Syftet med webbplatsen är att visa, lägga till och ta bort arbetserfarenheter genom att kommunicera med ett API med hjälp av Fetch API i JavaScript.

Webbplatsen är uppdelad i tre undersidor:

  • Start (index.html) – visar alla lagrade arbetserfarenheter från databasen
  • Lägg till (add.html) – möjlighet att lägga till nya arbetserfarenheter
  • Om sidan (about.html) – information om webbplatsen och dess syfte

Syfte

Syftet med webbplatsen är att fungera som ett gränssnitt mot ett REST-baserat API där arbetserfarenheter kan visas, skapas och raderas. Webbplatsen är byggd för att demonstrera hur en frontend-applikation kan kommunicera med en backend via HTTP-förfrågningar med hjälp av Fetch API.

Projektet syftar också till att ge förståelse för hur CRUD-operationer (Create, Read, Delete) implementeras i en modern webbapplikation samt hur data skickas och tas emot i JSON-format mellan klient och server.

Slutsatser

Under arbetet har jag fått en bättre förståelse för hur frontend och backend samverkar genom ett REST API. Jag har lärt mig hur man strukturerar en webbtjänst med Express samt hur man kopplar den till en databas. Jag har också förstått vikten av validering både på klient- och serversidan för att säkerställa korrekt datahantering. Fetch API har varit en stor del i projektet och har gett mig praktisk erfarenhet av hur moderna webbapplikationer kommunicerar med externa tjänster.
